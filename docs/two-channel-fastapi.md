# Two-channel honesty check — FastAPI (third-party)

A hitgate-native, **third-party** measurement of the self-indexed caveat. We run the *same*
retriever over the *same* corpus (FastAPI v0.115 `fastapi/` package) with two different
query sources, and put the numbers side by side:

- **auto-mined** — queries generated by `hitgate/generate.py` from the indexed corpus
  itself (the self-indexed, "label-free" default path).
- **hand-labeled NL** — 25 natural-language queries a human wrote
  (`hitgate/golden.fastapi.jsonl`), e.g. *"API key authentication using a query parameter"*.

FastAPI is the one corpus in the benchmark set that is **not** the author's own code
(see [SWEEP.md](./SWEEP.md)), so this is a bias-free check.

## Result (same engine, same corpus, same session)

| channel | n | Hit@1 | Hit@3 | Hit@5 | MRR |
|---|---|---|---|---|---|
| auto-mined (synthetic) | 42 | 0.595 | 0.976 | **1.000** | 0.764 |
| hand-labeled NL (human) | 25 | 0.640 | 0.920 | **0.920** | 0.760 |

## What it says — read both directions

**The synthetic channel is optimistic — that's the caveat, measured.** At Hit@5 the
auto-mined queries score **1.00 vs 0.92** for real human NL (+8pp). Self-indexed queries
are easier to retrieve because they are derived from the indexed text. This is exactly why
**a high self-indexed Hit@K is not evidence of relevance** — surface it for regression
detection, not absolute quality.

**But the gap is modest on a clean corpus — don't over-claim it either.** At Hit@1 and MRR
the two channels are effectively equivalent (auto is actually *lower* at Hit@1: 0.595 vs
0.640). The dramatic gaps you may have heard (toward ~0.27 Hit@5) come from *broad* or
*homogeneous* corpora, **not** a clean, well-structured third-party one like FastAPI. The
synthetic–human gap is **real and directional, but corpus-dependent in magnitude.**

**The circularity also shows up as query *quality*, not just score.** Raw `generate.py`
output is degenerate — it leaks source tokens and comments verbatim (e.g. literal
`param — TODO: update when deprecating Pydantic v1, import these types`, mined straight
from `params.py`). The auto channel above was **curated**: deduped, comment-leakage
dropped, capped at 3 cases per file (173 → 42). Uncurated, the self-indexing is even more
optimistic. A credible synthetic *natural-language* channel needs LLM naturalization
(`generate.py --llm`; deferred, see [ADR-0012](./adr/0012-caveat-investment-timing.md)).

## Caveats on the numbers themselves

- Numbers are from a **local (macOS)** run; the CI-frozen labeled baseline
  (`hitgate/baseline.fastapi.json`, Linux) is Hit@5 **1.0**. The ~8pp local↔CI offset is the
  documented cross-platform variance the eval-gate CI was Linux-frozen to remove. **What
  matters here is the within-session delta between the two channels**, both run on the same
  machine, not the absolute value.
- The auto channel is **identifier/concept** queries, not full sentences; that is the
  honest nature of distinctive-token mining.

## Reproduce

```bash
# index a third-party corpus (FastAPI v0.115 fastapi/ package)
DEST=/tmp/fastapi-v0115
git clone --depth 1 --branch 0.115.0 https://github.com/fastapi/fastapi.git "$DEST"
export RAG_INDEX_DIR="$DEST/.rag-index" RAG_SOURCE_ROOTS="$DEST/fastapi"
python -m ragcore.build

# synthetic channel: auto-mine, curate, run
python -m hitgate.generate --output hitgate/golden.fastapi-auto.jsonl   # then curate
RAG_RERANK_AUTO=off python -m hitgate.run --dataset hitgate/golden.fastapi-auto.jsonl --label fastapi-auto

# human channel: hand-labeled NL queries
RAG_RERANK_AUTO=off python -m hitgate.run --dataset hitgate/golden.fastapi.jsonl --label fastapi-labeled
```

The curated synthetic golden is committed at `hitgate/golden.fastapi-auto.jsonl` (42 cases).
